# Restitution graphique (forcegraph + timeline de montage) — Plan d'implémentation

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implémenter la spec `docs/specs/2026-07-04-minerva-viz-design.md` : une sous-commande `minerva viz` qui exporte une page HTML autonome (forcegraph des entités via force-graph vendoré + timeline de montage en SVG vanilla, slider temporel, couplage, tooltips, filtres).

**Architecture:** Toutes les transformations se font en Python dans `viz.py` (payload prêt à rendre : pistes fusionnées, écarts quantifiés, visibilité par moment) ; le JS de la page ne fait que du rendu. Assets (template + lib vendorée) dans le package, lus via `importlib.resources`, injectés par remplacement de placeholders — stdlib pur.

**Tech Stack:** Python 3.12, pytest ; force-graph (JS vendoré, MIT) ; SVG vanilla pour la timeline. Aucune dépendance Python nouvelle.

**Branche :** `viz` (déjà créée, spec commitée).

**Commandes :** tests = `.venv/bin/python -m pytest tests/ -q` ; un test ciblé = `.venv/bin/python -m pytest tests/test_viz.py::test_x -v`.

**Sémantique de visibilité (vérifiée dans le source, précise la spec) :** `entity_state`/`relation_state(at=M)` retournent les *attributs* connus à M, pas la visibilité. La visibilité au moment M se calcule par **premier rang** : rang d'un moment = `resolved_order` ; assertion/apparition non datée = rang −1 (« avant tout », convention de `_moment_ranks`, `model.py:321`). Une entité est visible à M si son premier rang ≤ rang(M) ; une relation si `max(son premier rang, rang de ses deux extrémités) ≤ rang(M)` (pas de lien pendouillant). Entité/relation sans aucune trace datée → rang −1, visible partout.

**Sécurité du template :** les données du payload viennent d'une extraction LLM — non fiables. Invariant : **toute valeur dynamique insérée dans du HTML passe par `esc()`** (échappe `&`, `<`, `>`) ; `innerHTML` n'est utilisé que sur des gabarits dont les interpolations sont toutes échappées. Les textes SVG passent par `textContent` (sûr par construction).

"""Restitution graphique : payload prêt à rendre + page HTML autonome.

Toutes les transformations (pistes, écarts, visibilité par moment) se font
ici, en Python — le modèle fait foi ; le JavaScript de la page ne fait que
du rendu. Visibilité par premier rang : rang d'un moment = resolved_order,
trace non datée = -1 (« avant tout », même convention que _moment_ranks)."""

from __future__ import annotations

import json
from importlib.resources import files

from .model import KnowledgeGraph


def build_payload(graph: KnowledgeGraph) -> dict:
    graph.timeline.resolve()
    moments = sorted(graph.timeline.moments, key=lambda m: m.resolved_order or 0)
    rank = {m.id: m.resolved_order or 0 for m in moments}

    rel_asserts: dict[tuple[str, str, str], list[dict]] = {}
    for a in graph.assertions:
        if not a.entity and a.relation_name and a.attribute:
            key = (a.relation_name, a.relation_source, a.relation_target)
            rel_asserts.setdefault(key, []).append(
                {"attribute": a.attribute, "value": a.value, "moment_id": a.moment_id}
            )

    gaps: dict[str, float] = {}
    for c in graph.timeline.constraints:
        if c.gap.days is None or c.source_id not in rank or c.target_id not in rank:
            continue
        if rank[c.target_id] == rank[c.source_id] + 1:
            gaps[str(rank[c.source_id])] = c.gap.days

    return {
        "entities": [
            {"name": e.name, "type": e.type, "aliases": e.aliases,
             "attributes": e.attributes}
            for e in graph.entities
        ],
        "relations": [
            {"name": r.name, "source": r.source, "target": r.target,
             "attributes": r.attributes,
             "assertions": rel_asserts.get((r.name, r.source, r.target), [])}
            for r in graph.relations
        ],
        "moments": [
            {"id": m.id, "order": rank[m.id], "days": m.resolved_days,
             "summary": m.summary}
            for m in moments
        ],
        "gaps": gaps,
        "tracks": [],   # Task 3
        "states": [],   # Task 4
    }
```

def _runs(orders: list[int]) -> list[list[int]]:
    """Fusionne une liste triée d'ordres en runs consécutifs [début, fin]."""
    runs: list[list[int]] = []
    for o in orders:
        if runs and o == runs[-1][1] + 1:
            runs[-1][1] = o
        else:
            runs.append([o, o])
    return runs
```

```python
def render_html(payload: dict) -> str:
    """Assemble la page autonome : template + lib vendorée + données.

    Le JSON est inséré dans un <script> : les séquences `</` sont échappées
    en `<\\/` pour qu'une valeur contenant `</script>` ne ferme pas le tag."""
    assets = files("minerva").joinpath("viz_assets")
    template = assets.joinpath("template.html").read_text(encoding="utf-8")
    lib = assets.joinpath("force-graph.min.js").read_text(encoding="utf-8")
    data = json.dumps(payload, ensure_ascii=False).replace("</", "<\\/")
    return template.replace("__FORCE_GRAPH_JS__", lib).replace("__MINERVA_DATA__", data)
```
